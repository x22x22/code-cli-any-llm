Usage: gal [command]

Gemini Any LLM AI 助手工具 - 通过本地网关提供优化的 AI 对话体验

Commands:
  code [prompt]     与 AI 助手对话（主要功能）
                    • 直接发送问题或任务给 AI 助手
                    • 自动启动和管理后台网关服务
                    • 支持所有 gemini CLI 原生参数和选项
                    • 首次使用会自动引导配置认证信息
                    • 若检测到新版本会先提示是否升级

  auth              配置 AI 服务认证信息
                    • 交互式设置 API Key、服务地址和默认模型
                    • 配置文件：~/.gemini-any-llm/config.yaml
                    • 首次使用或更换服务商时需要运行

  update            主动检查并安装最新版本
                    • 同步刷新版本缓存并执行升级命令
                    • 如果已是最新版本则提示后退出

网关管理命令:
  start             启动后台网关服务
                    • 手动启动网关，通常不需要直接使用
                    • 智能检测现有进程，避免重复启动
                    • 等待健康检查通过后返回

  stop              停止后台网关服务
                    • 优雅关闭网关进程和清理资源
                    • 等待进程完全退出（10秒超时）

  restart           重启网关服务
                    • 完整的停止和启动流程
                    • 配置更改后建议重启

  status            查看网关运行状态
                    • 显示服务健康状态和连接信息
                    • 显示进程信息和提供方配置

  kill              强制终止异常进程（故障排除）
                    • 多种方式查找和终止僵尸进程
                    • 仅在网关无响应时使用

  version           查看当前 gal 版本
  -h, --help        查看此帮助信息
  -v, --version     查看当前 gal 版本

配置与环境:
  • 配置文件：~/.gemini-any-llm/config.yaml
  • 默认网关端口：23062（可在配置文件中修改）
  • PID 文件：~/.gemini-any-llm/gateway.pid.json
  • 关闭自动更新检查：设置环境变量 GAL_DISABLE_UPDATE_CHECK=1

使用示例:
  # 首次使用，配置认证信息
  gal auth

  # 与 AI 助手对话（主要使用方式）
  gal code "请用TypeScript写一个HTTP服务"
  gal code "解释一下这段代码的作用"
  gal code "帮我优化这个算法"
  
  # 查看 gemini CLI 的更多选项
  gal code --help
  
  # 高级管理（通常不需要）
  gal status          # 查看服务状态
  gal restart         # 配置更改后重启
  gal kill           # 故障排除

故障排除:
  • AI 助手无响应：使用 'gal kill' 清理进程后重试
  • 认证失败：运行 'gal auth' 重新配置 API Key
  • 服务启动失败：检查网络连接和端口占用情况
  • 配置问题：检查 ~/.gemini-any-llm/config.yaml 文件
  • 权限问题：确保有读写 ~/.gemini-any-llm/ 目录的权限
